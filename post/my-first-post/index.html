Although I learned the course materials really well, this semester I got really terrible grades. Since then I realized that I should practice more to learn better. Tortuous building process

It took me no more than 20 minutes to make the decision that using hugo to build my blog due to itâ€˜s fast and convenient. Well, the download is indeed easy since we just use one command. However, to connect my local pc with github pages take a long time. I list some important processes and the problems I have met as follows:

  • I chose to use hugo-theme-jane to stylish my blog and I modified the font style by myself.

  • After successfully building my local server, I couldn’t wait to try my first post. Since I knew little about the Front Matter, so I copied the post from xianmin’s blog but found endless “?”. Thus I modified the language setting in the config.toml but nothing changed. Afterwards, I found that the encoding mode of the .md is ISO 8859-2, not utf-8. -> Take care when directly copy sth. from github.

  • I want my blog can be deployed automatically so I choose Travis CI and followed the steps in this article but I always failed to build. It took me a lot of time dealing with git command (e.g. delete the submodule configuration, remove all the files, …) to realize the correct setting. To build sucessfully, we should both configure the right .travis.yml and have a right configuration of articles. Really painful lessons!

  • I can’t use Chinese in my categories.

  • Build for many times but the github pages still can’t see the articles. I found it’s the building process to blame, but I can’t find the reason. When I change my front matter to the default, the building process finally can generate post directory. (Why?)
